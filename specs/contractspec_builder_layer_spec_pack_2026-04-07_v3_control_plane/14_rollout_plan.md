# Rollout Plan

## Phase 0: Core contracts

Build:
- `lib.builder-spec`
- `lib.provider-spec`
- Builder workspace, source, directive, runtime, provider, receipt, and readiness contracts

Exit criteria:
- contracts compile,
- sample manifests exist,
- replay and approval types are stable.

## Phase 1: Managed Builder MVP

Build:
- web chat guidance,
- document / ZIP ingestion,
- Studio snapshot import,
- source fusion,
- blueprint editor,
- provider routing policy,
- one conversational provider,
- one coding provider,
- one STT provider,
- managed preview path,
- readiness screen.

Exit criteria:
- non-engineer can create a draft app through managed runtime,
- delegated provider work is normalized and receipted,
- export is still governed.

## Phase 2: Mobile parity

Build:
- Telegram channel integration,
- WhatsApp channel integration,
- mobile review pages,
- voice-note flows,
- mobile parity harness suite,
- deep-link approval sessions.

Exit criteria:
- core authoring, review, and approval flows work from mobile,
- no critical builder workflow is desktop-only,
- evidence and blockers are visible on mobile.

## Phase 3: Local runtime

Build:
- local runtime registration,
- capability handshake,
- local runtime health and trust views,
- local preview/export path,
- runtime compatibility harness suite.

Exit criteria:
- power users can connect and use a local runtime,
- mobile operators can still observe and approve safe local actions,
- local and managed stay under the same Builder contracts.

## Phase 4: Hybrid runtime + provider comparison

Build:
- hybrid bridge,
- provider comparison mode,
- fallback and escalation policies,
- richer runtime sync controls.

Exit criteria:
- sensitive workflows can keep data local while preserving hosted control,
- high-risk tasks can use provider comparison and evidence-backed arbitration.

## Phase 5: Template and app-class expansion

Build:
- app templates,
- guided app-class packs,
- stronger connector packs,
- lower-friction onboarding for mainstream users.

Exit criteria:
- Builder feels like a product that creates useful apps,
- not a lab bench for people who enjoy configuration as a lifestyle.

## Sequencing advice

Do not start with:
- a dozen providers,
- fancy local hardware stories,
- or channel gimmicks.

Start with:
- contracts,
- receipts,
- runtime targeting,
- mobile-safe review,
- and one clean path from intent to verified preview.
