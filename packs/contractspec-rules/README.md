# contractspec-rules

This pack contains ContractSpec-specific governance for specification-first delivery, AI agent behavior, and docs discipline.

## Scope

- Rules that define ContractSpec process guardrails for contracts, planning, completion, docs, and AI governance.
- Lifecycle governance that keeps ideation -> planning -> shipping -> QA -> observability -> documentation coherent.
- Commands and agents for AI audit and implementation planning workflows.
- Skill support for creating new ContractSpec specifications.
- MCP integrations for ContractSpec internal, docs, and CLI services.

## Belongs here

- Policy that is unique to the ContractSpec product and platform.
- Guidance enforcing spec-first and governance workflows.

## Does not belong here

- Generic software engineering quality standards (use `software-best-practices`).
- Repository architecture and workspace context rules (use `workspace-specific`).
