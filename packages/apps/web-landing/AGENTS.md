# AI Agent Guide — `@contractspec/app.web-landing`

Scope: `packages/apps/web-landing/*`

Marketing landing page and documentation site for ContractSpec. Next.js application consuming `bundle.marketing` and `bundle.library`.

## LLM/Agent entry points

- **`/llms`** (aliases: `/llms.txt`, `/llms.md`) — Global monorepo summary; static file at `public/llms.txt`
- **`/llms-full.txt`** (alias: `/llms-full`) — Full content: all package READMEs aggregated; generated by `bun run llms:generate`
- **`/llms/[slug]`** — Per-package guide. Slug = package name without @contractspec (e.g. `lib.ai-agent`, `bundle.library`). Returns README as text/plain.
- **Subdomain**: `llms.contractspec.io` → serves `/llms.txt`
- AI consumers: read `public/llms.txt` or fetch `https://www.contractspec.io/llms`

## Quick Context

- **Layer**: app (Next.js)
- **Consumers**: end users (public website), AI agents, LLMs

## Architecture

- `src/app/(landing-marketing)/` — Marketing pages
- `src/app/(docs)/` — Documentation pages
- `src/app/(operate)/` — Operate/PM workbench
- `src/app/(sandbox)/` — Sandbox demos

| Route group | Paths | Purpose |
|-------------|-------|---------|
| `(landing-marketing)` | `/`, `/product`, `/templates`, `/contact`, `/changelog`, `/registry/*` | Marketing, product, templates |
| `(docs)` | `/docs/*` | Documentation |
| `(operate)` | `/operate/*` | PM workbench |
| `(sandbox)` | `/sandbox` | Sandbox demos |

## API surfaces

- `GET/POST /api/mcp` — MCP server for docs (Elysia handler)

## Guardrails

- Keep this app thin — page content and components come from `bundle.marketing` and `bundle.library`.
- No raw HTML — use Design System components.
- Uses Tailwind CSS v4 with PostCSS — do not introduce competing CSS solutions.
- Route changes affect SEO and external links; coordinate with marketing.
